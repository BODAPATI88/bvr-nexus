Platform Products

BVR Nexus ships as an integrated platform. Every component is designed to work together while remaining independently replaceable.

Core Platform

BVR API Gateway
v2.0.0

FastAPI-based gateway handling authentication (JWT + Keycloak), rate limiting, event validation, and the full platform registry. The single entry point for all platform operations.

FastAPI JWT Auth Rate Limiting PostgreSQL
AI Gateway
v2.0.0

Capability-based LLM routing across Claude, GPT, Kimi, and Ollama. Circuit breakers, Redis caching, cost tracking, and automatic failover. Provider priority declared in constitution.yaml.

Claude GPT-4 Kimi Ollama
BVR Workers
v2.0.0

Async Python workers that execute all business logic via BVR SDK. Ships with Review (repository analysis), Research (topic synthesis), and Achieve (goal-oriented task execution) workers.

Review Worker Research Worker Achieve Worker
BVR SDK
v2.0.0

Unified Python SDK for all platform operations. Covers events, AI calls, storage, telemetry, retry, policy, registry, and plugin discovery. Workers never reach infrastructure or providers directly.

ai_gateway_call emit_event upload_artifact policy_check

Orchestration & Governance

Kestra Workflows

Declarative YAML orchestration. Webhooks, retries, approvals, SLA gates, and notifications — zero business logic. Three bundled workflows: review, research, achieve.

OPA Governance

Three Rego policy files enforced on every event: RBAC (bvr.rego), cost guardrails (cost.rego), data residency (data_residency.rego).

Plugin System

Self-contained integration manifests under plugins/. Bundled: Claude (AI), GitHub (Code), Slack (Productivity). Auto-discovered at worker startup.
