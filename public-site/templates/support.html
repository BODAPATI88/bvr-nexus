Frequently Asked Questions

How do I add a new AI provider?

Add a provider block to contracts/constitution.yaml under the relevant capability, then configure the provider in the AI Gateway and add the API key to Vault. Workers do not need to change — the Capability Matcher selects the provider at runtime.

How do I create a new integration?

Create a plugin directory under plugins/<category>/<name>/ with six files: manifest.yaml, schema.yaml, permissions.yaml, health.py, worker.py, and __init__.py. The plugin is auto-discovered at worker startup.

What happens when an AI provider is unavailable?

The AI Gateway's circuit breaker detects failures and routes to the next provider in the fallback chain: Claude → GPT → Kimi → Ollama (local). Ollama always serves as the last-resort provider with no cost. Workers never implement fallback logic themselves.

How are costs controlled?

OPA enforces hard limits via cost.rego: $5.00 max per workflow execution, $50.00 max per user per day. Token usage is tracked per provider call in the AI Gateway via Redis counters. Events that would breach the limit are denied before execution.

Where are secrets stored?

All secrets are managed by HashiCorp Vault (KV v2). No credentials are stored in the application code, environment files, or Kubernetes manifests committed to git. Use bash k8s/create-secrets.sh to provision the K8s Secret from a local .env file.

Need help?

For platform issues, check the Operations Console first. For bugs or feature requests, open an issue on GitHub.
